# 分类体系图段落解读示例

## 示例：大模型高效微调方法分类

### 图 1: 大模型高效微调方法分类体系

![分类体系图](./figures/figure-1.jpg)

图 1 提出了大模型高效微调（Parameter-Efficient Fine-Tuning, PEFT）方法的分类体系，将现有方法划分为三大类：基于参数冻结的方法（如 Adapter、Prompt Tuning）、基于参数共享的方法（如 LoRA、BitFit）以及基于参数压缩的方法（如量化感知训练、知识蒸馏）。该分类的核心维度是"参数修改范围与侵入性"——从左到右，方法对预训练参数的侵入性逐渐增强，但所需的额外存储和计算开销也相应降低，这一轴线设计使得不同方法在"灵活性-效率"权衡空间中的位置一目了然。作者将 Prompt Tuning 和 P-Tuning 归为同一子类"软提示方法"，这一归类基于两者均通过优化连续向量引导模型行为，但未涵盖 2023 年下半年兴起的混合方法（如 Adapter + LoRA 的组合策略、MoE-PEFT 路由机制），这一遗漏可能反映了该综述的截稿时间较早，也提示分类体系在应对"方法组合化"趋势时存在扩展性不足。此外，分类体系中每个节点都标注了代表性工作的发表年份，清晰展示了该领域从 2019 年 Adapter 到 2023 年 LoRA 的技术演进脉络——早期工作聚焦于"在哪里插入额外参数"，后期工作则转向"如何更高效地参数化修改"，这一焦点转移反映了社区对 PEFT 本质理解的深化。

---

## 要点解析

**为什么这是好的段落解读：**

1. **分类逻辑解释**：不仅列举类别，还解释分类维度（"参数修改范围与侵入性"）和各类别在"灵活性-效率"权衡空间中的关系，使读者理解分类的设计意图而非仅知结果
2. **完整性评估**：指出遗漏（混合方法、MoE-PEFT 路由机制），并分析遗漏的可能原因（截稿时间）和对分类体系的影响（扩展性不足），体现批判性视角
3. **演进脉络**：通过年份标注说明技术发展趋势，并进一步提炼出焦点转移（"在哪里插入"→"如何更高效地参数化"），展现领域理解的深度
4. **综述类特定**：没有涉及伪代码或实验细节，聚焦分类体系本身的设计逻辑、完整性和演进趋势，符合综述类论文的解读重心
5. **术语规范**：PEFT、Adapter、LoRA 等术语保留英文，首次出现给出全称

**避免的写法：**

```markdown
❌ 不好的写法：
- 类别 1: 参数冻结方法 (Adapter, Prompt Tuning)
- 类别 2: 参数共享方法 (LoRA, BitFit)
- 类别 3: 参数压缩方法 (量化, 蒸馏)
- 遗漏: 混合方法
- 时间线: 2019-2023
```

这种列表化写法的问题在于：仅罗列分类结果而未解释分类维度的设计逻辑、未分析各类别间的权衡关系、未指出遗漏对分类体系扩展性的影响、未提炼技术演进的焦点转移，读者无法获得对领域结构的深层理解。
