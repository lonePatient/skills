## 图表解读规范

**核心原则**：以通顺段落叙述为主，信息点自然融入句子，避免结构化列表。

**呈现格式**：
```markdown
### 图 X: [图片标题]

![图X描述](./figures/figure-X.jpg)

[150-300 字通顺段落，涵盖以下信息点，但以自然叙述方式呈现：]
[ - 图表类型与核心内容概述 ]
[ - 关键数据点或趋势（如有具体数值，自然嵌入句子） ]
[ - 与论文主张的关联 ]
[ - 潜在问题或值得注意的细节（如有） ]

[对于复杂架构图，可分 2-3 个短段落，每段落聚焦一个层次或模块]
```

**段落写法要点**：
1. **数据点自然嵌入**："收敛至 0.85 附近"而非单独列数值
2. **趋势连贯描述**："前 10k 步快速下降，随后进入缓慢收敛阶段"
3. **关联自然提及**：在段落中自然说明图表支持了哪个论点
4. **审慎观察收尾**："不过，作者未报告标准差，单次实验的结果可靠性有待验证"

**禁止写法**：
```markdown
❌ 避免：
- 核心信息: ...
- 详细解读: ...
- 与论文主张的关联: ...
- 潜在问题: ...
```
