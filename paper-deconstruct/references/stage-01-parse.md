### 阶段 1：论文解析

深度分析之前，先提取并编目论文结构。**必须满足最低细节要求——浅层完成不可接受。**

保存为 `01-parse.md`：

```markdown
## 元数据
- 标题：[论文标题原文]
- 作者：[作者列表]
- 机构：[机构中文译名，格式："英文 (中文)"]
- 来源：[arXiv/会议名称]
- 链接：[链接，未知则填 "unknown"]
- 开源代码：[GitHub 链接，未开源则填 "未开源"]

## 类型判定
- 类型：[架构创新/实验评测/综述/理论证明/观点论述/系统工具/数据集]
- 类型特定焦点：[根据类型自动确定]
- 工程深度：[必须/可选/禁止] ← 由类型自动推导
- 领域：[CV/NLP/RL/多模态/等]
- 核心任务：[具体任务]

### 论文类型决策树

| 类型 | 核心判定信号 | 分析重心 | 工程深度 | 图片重点 |
|------|-----------|---------|------------------|---------|
| **架构创新** | 新模型/新算法/新训练策略 | 技术拆解 + 复现指南 | **必须** | 架构图、训练曲线、消融实验 |
| **实验评测** | 新数据集/新评估协议/大规模对比 | 实验设计分析 + 公平性审查 | **必须** | 对比表格、性能曲线、数据分布 |
| **综述** | 多工作梳理/分类体系/趋势分析 | 领域地图 + 技术演进脉络 | **禁止** | 分类体系图、时间线、对比表格 |
| **理论证明** | 定理/引理/收敛性分析 | 证明思路 + 假设条件 + 适用范围 | **禁止** | 证明流程图、收敛示意图 |
| **观点论述** | 新范式/批判现有方法/未来方向 | 论证逻辑 + 证据充分性 + 反方观点 | **可选** | 论证结构图、趋势预测图 |
| **系统工具** | 开源框架/推理引擎/训练平台 | 系统设计 + 性能优化 + 使用指南 | **必须** | 系统架构图、性能对比图、使用流程 |
| **数据集** | 新数据收集/标注/质量分析 | 构建流程 + 质量控制 + 使用注意事项 | **可选** | 数据样本、分布统计、标注界面 |

**类型判定辅助信号**（边界模糊时参考）：
- Method 章节页数占比 > 40% → 倾向架构创新
- Related Work 章节页数占比 > 30% → 倾向综述
- Theorem/Lemma/Proof 出现频率 > 5 次 → 倾向理论证明
- 无 Experiments 章节 → 倾向理论证明或观点论述
- 提供 GitHub 链接 + API 文档 → 倾向系统工具
- 数据集描述页数 > 50% → 倾向数据集

## 核心主张识别
- 核心技术主张：[论文核心声称]
- 关键支撑实验：[支撑声称的关键实验]
- 最重要 baseline 对比：[最重要的 baseline 对比]

## 结构图
- [章节列表，注明行号/页码范围]
- [仅 Method 与 Experiments 章节的子节列表]

## 图片清单

| 图号 | 类型 | 内容摘要 | 重要性 | 关联章节 |
|------|------|---------|--------|---------|
| Fig 1 | [架构图/曲线/表格/示意图/样本] | [一句话描述核心内容] | [P0/P1/P2] | [引言/方法/实验/附录] |

**重要性分级**：
- **P0 (核心图)**: 直接影响对方法/实验的理解，必须详细段落化解读
- **P1 (辅助图)**: 帮助理解背景或展示效果，适度段落化解读
- **P2 (补充图)**: 提供额外信息，简要提及即可

## 代码可获取性
- [列出可用脚本、位置及其与核心方法的相关性]
```

**最低细节强制要求：**
- 结构图必须为 Method 与 Experiments 章节细化到子节。
- 图片清单对每个图片的内容描述必须至少包含一句完整的话。
- 若某字段确实无法解析，写 "无法解析" 并说明原因，不能留空。
- **PDF 格式图片必须标记为待转换为 JPG/PNG。**
