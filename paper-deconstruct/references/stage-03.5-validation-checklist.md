### 阶段 3.5：批判前自检

快速检查初稿的结构完整性与流程合规性，**不是内容质量审校**（内容质量由阶段 4 负责）。

保存为 `03.5-validation.md`：

```markdown
## 结构完整性检查
- [ ] 03-draft.md 是否包含所有必需章节（按类型适配）？
- [ ] 每个核心模块是否都有"功能职责+输入输出+算法流程"？
- [ ] 复杂度判定是否与内容匹配？
- [ ] 知识补充是否已触发并保存到 03-research-notes.md？

## 类型适配检查
- [ ] 论文类型判定是否合理？（复查判定信号）
- [ ] 是否包含该类型禁止输出的内容？（如综述出现伪代码）
- [ ] 是否遗漏该类型必须输出的内容？（如综述缺少分类体系）
- [ ] Engineering Depth 标注是否与类型一致？

## 图片覆盖检查
- [ ] 论文中所有 P0 级图片是否已引用并附段落解读？
- [ ] 每张图片是否有对应的 Markdown 图片链接且可访问？
- [ ] 图片解读是否为通顺段落而非碎片化列表？
- [ ] 是否出现"如图X所示"但无实际解读的情况？
- [ ] 复杂架构图是否提供了分层的文字解释？

## 批判性思维完整性检查（新增）
- [ ] §9 最脆弱假设是否找到了失败代价最大的假设？（非泛泛列举局限性）
- [ ] §10 最小复现实验是否真的可在 1 周内完成？数据/实现/测量/判定标准是否清晰？
- [ ] §11 最强反例是否提出了具体替代解释或可预测失败条件？（非泛泛攻击）
- [ ] §12 Follow-up 是否为非增量 idea？是否有新 framing/新目标/新连接？

## 信息来源纪律检查（新增）
- [ ] 每个关键 claim 是否标注了来源类别（论文声称/文献结论/合理推断/猜测）？
- [ ] 是否有把推断写成事实的情况？
- [ ] 是否有把猜测写成推断的情况？

## 循环控制
- **最大尝试次数**：2 次（首次 + 1 次补充）

**强制通过条件**：
- 若某模块信息论文本身未提供，标注"未提供"并说明理由 → 视为通过
- 若类型判定存在争议，记录判定依据 → 视为通过
- 若图片为 P2 级且未解读，记录原因 → 视为通过

**强制阻断条件**：
- P0 图片缺失解读 → 必须补充
- 类型适配严重偏差（如综述出现伪代码） → 必须修正
- 出现事实性错误或数据引用错误 → 必须修正
- §9-§12 内容空洞无实质 → 必须补充

## 通过/失败判定
- [ ] 通过：结构完整，进入阶段 4 批判
- [ ] 失败（第 1 次）：返回阶段 3 补充，记录补充计划
- [ ] 失败（第 2 次）：记录"已知局限"，强制进入阶段 4
```
